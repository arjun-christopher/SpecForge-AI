## Tips for Better Results

1. **Be Specific**: Include details about features, technologies, and requirements
2. **Mention Integrations**: Specify third-party services (Stripe, OAuth providers, etc.)
3. **Include Scale**: Mention if you need multi-tenant, real-time, or high-scale features
4. **Security Requirements**: Specify authentication, authorization, and compliance needs
5. **User Roles**: Define different user types and their permissions

## Prompt Template

Use this template for comprehensive results:

```
Build a [type of system] with [core features], [authentication method],
[data storage requirements], [third-party integrations], and [special requirements].
Include [specific functionality] and support for [user roles/permissions].
```

**Example**:

```
Build a task management system with project boards, task assignments, file attachments,
JWT authentication, PostgreSQL database, Slack integration, and email notifications.
Include time tracking functionality and support for Admin, Manager, and Team Member roles.
```
